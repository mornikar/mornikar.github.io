DRF框架就是RestFramework框架（rest_framework[导入的方式]）

  • REF开发模式： 前后端分离模式开发的
    • RESTful
  • 域名
    接口设计方法：【大型服务部署放在专有域名下（整个服务器提供此类服务）】

小型 子应用：

  • 版本（在app后面放版本号）

  • 路径(名词，复数)
    遵循restful风格：/books/

  • #请求动作

  • 没pk（id）：
    get + /books/ ==> 获取全部对象
    post + /books/ ==> 新建一数据

  • 有pk：
    get + /books/ ==> 获取单一对象
    put + /books/ ==> 更新单一对象
    delete + /bookds/ ==> 删除单一对象

  • 过滤参数
    过滤的参数一般是放在查询字符串中携带

  • 过滤的参数一般放在查询字符串中携带
    查询字符串 = page

  • 状态码
